# 🛠️ 项目排查与踩坑记录 (Troubleshooting)

在搭建和调试 AI Agent 自动化写入飞书工作流的过程中，遇到并解决了以下问题，特此记录以沉淀经验：

## 1. Coze 飞书插件首次授权报错 (401 Unauthorized)
- **现象**：在 Coze 工作流中添加飞书官方插件后，运行测试时节点报错，提示无权限。
- **排查**：检查后发现是因为飞书账号的安全设置拦截了外部应用的静默授权。
- **解决**：在弹出的飞书授权页面中，手动确认赋予该应用 `docx:document` (文档读写) 等必须的 API 权限，重新运行后解决。

## 2. 大模型输出的 Markdown 分镜表在飞书中格式错乱
- **现象**：Agent 生成了完美的 Markdown 表格，但通过 API 写入飞书云文档后，表格变成了普通纯文本。
- **排查**：飞书旧版云文档 API 对复杂 Markdown 表格的支持存在解析兼容性问题。
- **解决**：在 System Prompt 的最后强调“输出标准的纯文本或基础 Markdown”，并确保 Coze 中选择的插件是支持最新版飞书文档（docx）的节点。

## 3. 违禁词质检导致模型陷入死循环
- **现象**：早期测试时，由于设定的违禁词过于严格（如直接封禁了“糖”字），导致模型在生成“0 蔗糖”时反复触发风控重写。
- **解决**：优化了 SKILL 里的质检规则，引入了“强制拦截黑名单”与“柔性替换建议”分层处理，并通过限定场景（如“控糖”仅在作为功效承诺时才拦截）解决了该 Bug。
