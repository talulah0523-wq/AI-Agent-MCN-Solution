# Skill · MCN Script Assistant
> 小红书博主风格短视频种草脚本生成器

## 一、适用场景

本 Skill 适用于以下任意一种业务场景：

- 品牌方需要 **基于已选定的对标博主** 批量生产小红书短视频脚本
- MCN 机构需要为旗下博主 **快速产出多个备选脚本** 供拍摄选用
- 投放团队需要在 **同一品牌、不同场景**（早餐 / 运动后 / 下午茶等）下做矩阵化内容铺设
- 产品上新期需要 **48 小时内出 10+ 条差异化脚本** 的高速场景

⚠️ 不适用场景：直播话术、长图文笔记、信息流硬广文案。

## 二、输入材料（用户需准备）

调用本 Skill 前请准备好以下 4 类资料，缺一项都会显著降低产出质量：

1. **品牌 Brief（必填）**
   - 品牌名、产品 SKU、核心卖点（≤3 个）、目标人群画像
   - 平台限制、**违禁词清单**（合规团队提供）

2. **对标博主拉片资料（必填）**
   - 博主人设一句话总结
   - 至少 1 条爆款笔记的逐字稿
   - 视觉结构拆解（开头/中段/结尾的镜头特征）
   - 种草逻辑总结（4 步法或同等颗粒度）

3. **场景定义（必填）**
   - 本次脚本要切入的具体使用场景（越具体越好）
   - 例：✅"健身房力量训练后 15 分钟内的蛋白补给" ❌"健身后吃"

4. **视频参数（选填，有默认值）**
   - 时长（默认 60–90s）、目标互动指标（默认收藏率优先）

## 三、执行步骤

1. Step 1  Brief 拆解         → 抽取卖点 / 人群 / 违禁词清单
2. Step 2  风格指纹提取       → 提炼对标博主的语言+视觉指纹
3. Step 3  三段式脚本生成     → 钩子 / 痛点 / 解决+升华
4. Step 4  违禁词强制质检     → 命中即回炉，直至全部通过
5. Step 5  Markdown 格式化输出 → 标题 + 口播稿 + 分镜表 + 质检报告

每一步的详细规则参见 `prompts/system_prompt.txt`。

## 四、输出格式

标准输出为一个 Markdown 文档，包含以下 4 个固定模块：

1. **标题区**：小红书风格爆款标题（带 1–2 个 emoji，≤20 字）
2. **完整口播稿**：博主可直接背诵的连贯文本
3. **分镜表**：Markdown 表格，至少 6 镜，列含【镜头序号 / 画面描述 / 时长 / 口播 / 产品植入点】
4. **风控质检报告**：明确列出已规避的违禁词类别

## 五、风险检查清单（每条脚本上线前必过）

- [ ] 未出现 **减肥类** 词汇（减肥/瘦身/掉秤/燃脂/暴瘦）
- [ ] 未出现 **功效承诺** 词汇（降糖/降血糖/治疗/根治/改善 XX 病）
- [ ] 未出现 **绝对化用语**（最 / 唯一 / 100% / 第一）
- [ ] 未出现 **医疗暗示**（药用 / 疗效 / 医生推荐 / 临床证明）
- [ ] 产品功效表达限定在"卖点描述"而非"效果承诺"
- [ ] 博主口吻一致性：是否符合目标博主的语言指纹
- [ ] 场景具体性：是否有可拍摄的画面颗粒度
- [ ] 节奏合理性：是否符合"快—慢—静"三段式节奏
- [ ] 产品出场时机：是否在 30s 之后（避免开门见山的硬广感）

## 六、兜底与异常处理机制（Fallback）

为保证自动化流水线的稳定性，Agent 需遵循以下异常处理逻辑：
1. **风控质检未通过**：触发自动回炉重写，最多重试 2 次。若 2 次后仍包含违禁词，中止输出并亮红灯提示“需人工介入修改”。
2. **飞书 API 写入失败/超时**：不影响脚本生成，前端直接返回 Markdown 文本代码块，并附带错误日志（如：飞书 Token 失效），供操作员手动复制。
3. **大模型生成超时**：若主流模型（如 GPT-4o）响应超过 60 秒，建议在 Coze 工作流中设置分支节点，自动降级切换至备用轻量级模型。
